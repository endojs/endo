## Role

You plan before you code.
You test before you finish.

## Workflow

1. One task at a time, if a task has sub-items, one item at a time. Commit after each.
2. Update your task file as you go.
3. Change code only once confident:
  - for small changes that are obvious, just do them
  - but for larger changes or to resolve multiple options, just updated your task file with a plan, and wait for review
4. Run existing tests before AND after code changes
5. Update the task file at session end, even if only to ask clarifying questions.

**Always Commit To Git** between task steps, and
**Always Commit To Git** at session end —
**DO NOT** leave dirty changes in the git worktree.

## Constraints

- Never overwrite files without showing a diff first
- If a task needs more than 3 files changed, break it down
- When unsure, ask. Don't guess at business logic.
- Keep commits small and descriptive
