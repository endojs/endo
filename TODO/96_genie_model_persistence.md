# Phase 2 sub-task: model config persistence

Implements § 3d of [`TODO/92_genie_primordial.md`](./92_genie_primordial.md).
Lands in parallel with sub-task 95 (`/model` builtin); 95's
`/model commit` calls into the helpers defined here.

## Goal

Land the `<workspace>/.genie/config.json` schema, the atomic
read/write helpers, and the boot-time precedence resolution so a
daemon restart re-enters piAgent mode without re-prompting the
operator for credentials.

## Files

- `packages/genie/src/primordial/persistence.js` — *new* module:
  - `loadConfig(workspaceDir): Promise<Config | undefined>`
  - `saveConfig(workspaceDir, config): Promise<void>`
  - `clearConfig(workspaceDir): Promise<void>`
  - Internal: `validateConfig(raw): Config | undefined` performs
    the schema check; logs and returns `undefined` on schema
    mismatch.
- `packages/genie/src/primordial/types.js` — *new* module
  exporting the `Config` JSDoc typedef so model-handler.js (sub-task
  95), the persistence module, and boot can share one type.
- `packages/genie/main.js` — replace the `// TODO(96)` stub
  introduced by sub-task 93 with a real call to `loadConfig` in the
  env-precedence block.
- `packages/genie/CLAUDE.md` — extend § "Env-var config" with a
  pointer to `.genie/config.json` and the precedence rule
  (env > persisted > primordial).

## Schema (v1)

```json
{
  "_README": "Generated by /model commit. Plaintext — see CLAUDE.md before checking in.",
  "version": 1,
  "model": {
    "provider": "anthropic",
    "modelId": "claude-sonnet-4-5",
    "credentials": {
      "ANTHROPIC_API_KEY": "sk-ant-..."
    },
    "options": {}
  }
}
```

- `version: 1` — every loader checks this; future migrations bump
  the version and add a `migrate` step.
- `credentials` — keyed by env-var name (matching the pi-ai
  `getEnvApiKey` convention) so the loader can stamp values into
  `process.env` for the agent pack to find.
- `options` — non-secret per-provider settings (e.g. `OLLAMA_HOST`).

## Implementation notes

- **Atomicity.** Write to `.genie/config.json.tmp`, `fsync`, then
  `rename` to `.genie/config.json`.  POSIX `rename` is atomic on
  the same filesystem.  On Windows, the same pattern works but the
  rename is best-effort; document the caveat.
- **File mode `0600`.** Use `fs.chmod` after the rename.  Fail
  loudly if the platform does not support it (Windows: log a
  warning, do not throw).
- **Boot-time stamping.** When `loadConfig` returns a config in
  `make()`, copy `config.model.credentials` into a process-local
  env shim (do *not* mutate `process.env` in the worker — that is
  shared with co-tenants).  Pass them through to
  `makeGenieAgents` via the per-agent options that already accept
  `model:` overrides; the model-string path remains
  `${provider}/${modelId}`.

  Actually, look closely at `packages/genie/src/agent/index.js:78`
  (`getOllamaApiKey`) and `pi-ai/dist/env-api-keys.js:47-105` —
  both read `process.env` directly.  The cleanest workaround for
  v1 is to set the env vars in `process.env` *just* for the agent
  construction window, then unset them.  Document the hack and
  track a follow-up for a credential-passing channel that does not
  go through `process.env`.

- **Sentinel handling.** Absent file → return `undefined`.
  JSON parse error → log a warning with the file path, return
  `undefined`.  Schema mismatch (missing `version`, wrong type) →
  log an error with the offending field, return `undefined`.
  Never throw out of `loadConfig`.

## Tests

- New file `packages/genie/test/primordial/persistence.test.js`:
  - Round-trip `saveConfig` then `loadConfig` returns the same
    config.
  - Atomic write: simulate a partial write by leaving a stray
    `.tmp` file and assert `loadConfig` still returns the previous
    good value.
  - Schema validation: corrupt JSON returns `undefined` and logs
    a warning.
  - File mode: on POSIX, the file is `0600` after `saveConfig`.
- Extend `packages/genie/test/boot/self-boot.test.js`:
  - Primordial-then-commit-then-restart: pre-seed
    `<workspace>/.genie/config.json`, boot a worker without
    `GENIE_MODEL`, assert `agent ready` log line and (via
    `/model show`) the loaded model is reported.

## Acceptance

- `npx ava packages/genie/test/primordial/ packages/genie/test/boot/
  --timeout=120s` passes.
- A worker booted from a pre-seeded `.genie/config.json` reaches
  `agent ready` without operator input.
