# Phase 2 sub-task: model config persistence

**Status:** landed.  See the bottom of this file for the implementation
notes that diverged from the original plan.

Implements ┬º 3d of [`TODO/92_genie_primordial.md`](./92_genie_primordial.md).

## Goal

Land the `<workspace>/.genie/config.json` schema, the atomic
read/write helpers, and the boot-time precedence resolution so a
daemon restart re-enters piAgent mode without re-prompting the
operator for credentials.

  - `loadConfig(workspaceDir): Promise<Config | undefined>`

  - `clearConfig(workspaceDir): Promise<void>`
  - Internal: `validateConfig(raw): Config | undefined` performs

## Schema (v1)

```json
{
  "_README": "Generated by /model commit. Plaintext ÔÇö see CLAUDE.md before checking in.",
  "version": 1,
  "model": {
    "provider": "anthropic",
    "modelId": "claude-sonnet-4-5",
    "credentials": {
      "ANTHROPIC_API_KEY": "sk-ant-..."
    },
    "options": {}
  }
}
```

- `version: 1` ÔÇö every loader checks this; future migrations bump
  the version and add a `migrate` step.
- `credentials` ÔÇö keyed by env-var name (matching the pi-ai
  `getEnvApiKey` convention) so the loader can stamp values into
  `process.env` for the agent pack to find.
- `options` ÔÇö non-secret per-provider settings (e.g. `OLLAMA_HOST`).

## Implementation notes

- **Atomicity.** Write to `.genie/config.json.tmp`, `fsync`, then
  `rename` to `.genie/config.json`.  POSIX `rename` is atomic on
  the same filesystem.  On Windows, the same pattern works but the
  rename is best-effort; document the caveat.
- **File mode `0600`.** Use `fs.chmod` after the rename.  Fail
  loudly if the platform does not support it (Windows: log a
  warning, do not throw).
- **Boot-time stamping.** When `loadConfig` returns a config in
  `make()`, copy `config.model.credentials` into a process-local
  env shim (do *not* mutate `process.env` in the worker ÔÇö that is
  shared with co-tenants).  Pass them through to
  `makeGenieAgents` via the per-agent options that already accept
  `model:` overrides; the model-string path remains
  `${provider}/${modelId}`.

  Actually, look closely at `packages/genie/src/agent/index.js:78`
  (`getOllamaApiKey`) and `pi-ai/dist/env-api-keys.js:47-105` ÔÇö
  both read `process.env` directly.  The cleanest workaround for
  v1 is to set the env vars in `process.env` *just* for the agent
  construction window, then unset them.  Document the hack and
  track a follow-up for a credential-passing channel that does not
  go through `process.env`.

- **Sentinel handling.** Absent file ÔåÆ return `undefined`.
  JSON parse error ÔåÆ log a warning with the file path, return
  `undefined`.  Schema mismatch (missing `version`, wrong type) ÔåÆ
  log an error with the offending field, return `undefined`.
  Never throw out of `loadConfig`.

## Implementation notes (post-landing)

- **Files added.**
  `packages/genie/src/primordial/types.js` (Config typedef),
  `packages/genie/src/primordial/persistence.js` (loadConfig /
  saveConfig / clearConfig / validateConfig + `CONFIG_REL_PATH` /
  `README_TEXT` constants), and a unit-test file
  `packages/genie/test/primordial/persistence.test.js`.
- **`main.js` boot precedence.**
  The `// TODO(96)` stub `loadConfig` was replaced by a direct call to
  the new helper.  When a persisted Config is present, the boot path
  stamps `model.credentials` and `model.options` into `process.env`
  *before* `makeGenieAgents` so pi-ai's request-time `getEnvApiKey`
  finds them.  The hack is documented inline (`stampPersistedEnv`) ÔÇö
  existing env values win over the persisted ones so launcher-supplied
  overrides are not silently clobbered.  pi-ai reads env vars at
  request time (see `pi-ai/dist/env-api-keys.js`), so the env stays
  set for the lifetime of the worker rather than being unset after
  construction; the cleaner channel is tracked under TODO/92 ┬º 3g.
- **CLAUDE.md.**
  `packages/genie/CLAUDE.md` ┬º "Env-var config" gained a
  "Persisted model config" subsection covering the file location, the
  env > persisted > primordial precedence rule, and the env-stamping
  hack.
- **Tests.**
  `npx ava packages/genie/test/primordial/ packages/genie/test/boot/
  --timeout=120s` ÔÇö 58 tests pass (13 new persistence tests + 1 new
  pre-seeded-config boot test alongside the existing suites).
  Persistence tests that need to inspect warning output run serially
  because they swap `process.stderr.write` for capture; SES freezes
  `console.warn` so direct stubbing is not an option.
- **Atomic-write shape.**
  `saveConfig` opens the temp file with mode `0600`, writes, `fsync`s,
  closes, then `rename`s into place; on POSIX a follow-up `chmod 0600`
  is best-effort (warn-only on Windows).  A stray `.tmp` is left
  untouched if the caller crashes mid-write ÔÇö the next `loadConfig`
  ignores it because the loader only reads the canonical path.
